| Area                       | Finding (What’s wrong)                   | Evidence (Why this matters)                                                                                                     | Recommendation (What to change)                                                                      |
| -------------------------- | ---------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------- |
| **Brand Name**             | “Dream Guardian” is abstract and unclear | Descriptive, short names are learned and recalled faster; non-descriptive names create cognitive load (NIH, HBS naming studies) | Rename to something functional and obvious (e.g., **BuildTogether**, **CoupleSaver**, **GoalCoach**) |
| **Logo**                   | Heart icon feels like a dating app       | Rounded/heart shapes signal romance & intimacy; finance products should signal trust & stability                                | Replace with **owl / shield / piggy-bank / upward path** icon                                        |
| **Color System**           | Red used heavily for primary actions     | Red in fintech is linked to errors, debt, urgency; reduces trust                                                                | Use **blue/green/purple** for primary CTAs; reserve red only for destructive actions                 |
| **Visual Consistency**     | Too many color styles across screens     | Consistent color usage improves brand recognition by up to 80%                                                                  | Lock **1 primary + 1 secondary + neutral greys** across entire app                                   |
| **Landing Page Copy**      | Users must read everything to “get it”   | Users scan in F-pattern; long paragraphs are skipped                                                                            | Rewrite into **headline → 3 bullets → 1 visual → CTA**                                               |
| **Screenshots on Landing** | Screenshots are dense and unlabeled      | Users don’t interpret UI screenshots without guidance                                                                           | Add **callout labels** (“Track”, “Split”, “Save”) on images                                          |
| **Onboarding Flow**        | No progress indicator                    | Progress indicators reduce anxiety & increase completion (NN Group)                                                             | Add **Step 1 of 3 / Step 2 of 3** indicator                                                          |
| **Camera Permission**      | Requested too early                      | Early permission requests increase drop-off                                                                                     | Let users explore first; ask permission *when needed*                                                |
| **Dashboard Density**      | Too much info above the fold             | Cognitive overload reduces comprehension                                                                                        | Surface **1 primary insight** per screen                                                             |
| **AI Feature Framing**     | “AI” feels abstract                      | Users trust AI more when examples are shown                                                                                     | Show **example prompts** (“Add grocery $42”)                                                         |
| **Goals (Dreams)**         | No target date or emotional framing      | Deadlines increase follow-through (goal gradient effect)                                                                        | Add **target date + short “why” field**                                                              |
| **Navigation**             | Some flows feel “trapped”                | Missing back navigation increases frustration                                                                                   | Always show a **Back / Close** option                                                                |
| **Settings Page**          | Long, flat list                          | Grouping improves scanability                                                                                                   | Group into **Account / Money / AI / Privacy** sections                                               |
